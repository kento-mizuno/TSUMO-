# 正しいファイル追加方法

## 問題

「Reference files in place」を選択すると、フォルダ参照（小さな矢印アイコン）になってしまいます。これは正しくありません。

## 正しい設定方法

### ステップ1: フォルダ参照を削除

1. 小さな矢印が表示されているフォルダを選択
2. 右クリック → `Delete`
3. **"Remove Reference"** を選択

### ステップ2: 正しい設定で追加

1. プロジェクトナビゲーターで `TSUMO` フォルダを**右クリック**
2. `Add Files to "TSUMO"...` を選択
3. フォルダを選択（例: `/Users/TSUMO!/TSUMO/ViewModels`）
4. **重要**: ダイアログで以下の設定を確認：

#### Action（アクション）セクション
- **"Copy items if needed"** のチェックを**外す**（ファイルは既に正しい場所にあるため）
- または、**"Reference files in place"** を選ばない（これがフォルダ参照になってしまう原因）

#### Groups（グループ）セクション
- **"Create groups"** が選択されていることを確認
  - これが重要！フォルダ参照ではなく、グループとして追加されます

#### Targets（ターゲット）セクション
- **"TSUMO"** にチェックを入れる

5. `Finish` をクリック

## 確認方法

正しく追加されると：
- ✅ フォルダアイコンに**小さな矢印がない**
- ✅ フォルダを展開すると、中身のファイルが表示される
- ✅ ファイルをクリックすると、エディタで開ける

間違って追加されると：
- ❌ フォルダアイコンに**小さな矢印がある**（フォルダ参照）
- ❌ フォルダを展開しても、中身が表示されない

## もし「Create groups」オプションが表示されない場合

Xcodeのバージョンや設定によって、ダイアログの表示が異なる場合があります。

### 代替方法: 個別にファイルを追加

フォルダ参照になってしまう場合は、フォルダ内のファイルを個別に追加する方法もあります：

1. フォルダ参照を削除
2. `TSUMO` フォルダを右クリック → `Add Files to "TSUMO"...`
3. フォルダ内の**全てのSwiftファイル**を選択（`Cmd + クリック`で複数選択）
4. 設定：
   - **"Copy items if needed"** のチェックを**外す**
   - **"Create groups"** を選択（フォルダ名でグループが作成される）
   - **"TSUMO"** にチェック
5. `Finish` をクリック

## 現在の状態を確認

以下のコマンドで、各フォルダ内のファイル数を確認できます：

```bash
# ViewModels
ls TSUMO/ViewModels/*.swift | wc -l  # 9ファイル

# Services
ls TSUMO/Services/*.swift | wc -l    # 1ファイル

# Utilities
ls TSUMO/Utilities/*.swift | wc -l   # 2ファイル
```

Xcodeで正しく表示されていれば、これらのファイルが全てプロジェクトナビゲーターに表示されるはずです。
